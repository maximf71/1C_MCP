# Импорт внешних обработок и отчётов

Loopback-мост поддерживает endpoint `/external/import-xml`. Он использует штатный API EDT `IImportOperationFactory.createImportExternalObjectOperation` и:

- создаёт или обновляет только проекты с префиксом `CodexExt_`;
- связывает внешний проект с зафиксированным базовым проектом конфигурации;
- принимает только относительный путь к корневому XML внутри `onec.mcp.externalRoot` (по умолчанию домашний каталог пользователя);
- отклоняет абсолютные пути, выход через `..`, символьные ссылки наружу и не-XML файлы;
- выполняет импорт в фоновом задании EDT с тайм-аутом 300 секунд;
- не обновляет информационную базу.

STDIO MCP-сервер дополнительно ограничивает диагностику и сборку теми же проектами `CodexExt_*`, а готовые EPF/ERF пишет только в `.edt-external-builds/<project>` внутри разрешённого корня.
