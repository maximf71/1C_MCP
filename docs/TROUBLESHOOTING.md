# Решение проблем

## 403 Forbidden

Проверьте, кто вернул 403. Для локальной схемы правильный адрес DitriX — `http://127.0.0.1:8765`; устаревший HTTP-контур базы на `localhost:8080` для чтения структуры EDT не нужен. Убедитесь, что в аргументах обёртки заданы оба параметра `--ditrix-edt-url` и `--ditrix-project`, затем перезапустите Codex.

## Connection refused на 8765

EDT не запущен, плагин не установлен, MCP Server остановлен в настройках или используется другой порт. Откройте настройки MCP Server в EDT и проверьте `/health`.

## Project is building / derived data not complete

EDT ещё строит модель. Обёртка автоматически повторяет только эти временные ошибки в течение 30 секунд. Если ошибка остаётся, откройте Problems, устраните ошибки импорта и дождитесь окончания фоновых задач.

## Инструменты не появились после обновления

Codex держит STDIO-процесс и схему инструментов в памяти. Полностью завершите Codex и запустите снова. Проверьте, что `command` указывает на новый EXE.

## Пустой или серый снимок формы

Добавьте `-DnativeFormBufferedLayoutRender=true` после `-vmargs` в `1cedt.ini` и полностью перезапустите EDT.

## Нет bridge.json

Опциональный bridge не стартовал. Проверьте:

- JAR лежит в `dropins\...\plugins`;
- в `1cedt.ini` задано непустое `-Donec.mcp.project`;
- каталог из `-Donec.mcp.runtimeDir` доступен для записи;
- EDT был полностью перезапущен;
- Error Log EDT не содержит ошибки `Cannot start Codex EDT bridge`.

`bridge.json` удаляется при штатном завершении EDT — это ожидаемо.
