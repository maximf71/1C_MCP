# Интеграция с DitriX EDT-MCP

STDIO-сервер объединяет два возможных EDT-бэкенда:

1. официальный сторонний DitriX EDT-MCP 2.8.1 по MCP-over-HTTP на `http://127.0.0.1:8765/mcp`;
2. опциональный локальный `edt-bridge`, предоставляющий дополнительные нативные инструменты проекта.

Go-сервер не копирует и не линкует код DitriX. Он динамически получает MCP-схемы и проксирует разрешённые инструменты, сохраняя исходное содержимое ответа, включая изображения.

## Границы безопасности

- URL должен использовать `http`, loopback-хост и путь `/mcp`; перенаправления отклоняются;
- каждый параметр `projectName` принудительно заменяется именем, заданным в `--ditrix-project`;
- аргументы произвольных каталогов отклоняются;
- управление жизненным циклом базы, учётными данными, импортом/экспортом в произвольные каталоги, Git-ветками и неизвестными будущими инструментами не проксируется;
- локальные инструменты с совпадающим именем имеют приоритет;
- операции записи и удаления сохраняют соответствующие MCP-аннотации.

## Запуск

В настройках DitriX EDT-MCP включите автозапуск и порт 8765. Запустите EDT, откройте целевой проект и дождитесь окончания построения.

```powershell
Invoke-RestMethod http://127.0.0.1:8765/health
```

Передайте точное имя проекта в `--ditrix-project`. После изменения регистрации или EXE полностью перезапустите MCP-клиент.

Используйте `get_ditrix_edt_capabilities`, чтобы увидеть версию upstream, закреплённый проект, зарегистрированные инструменты, локальные переопределения и исключения политики.
