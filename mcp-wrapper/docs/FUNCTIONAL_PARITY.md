# Functional parity map

## Public MIT level

- live metadata, object and form inspection;
- read-only queries and validation;
- event log and configuration information;
- subsystem topology (`orphans`, `containing`, `intersections`);
- smart, exact and regular-expression dump search with filters and persistent cache;
- file/server infobase extension installer with platform compatibility patching.

## Enhanced analogs

- offline XML validation and deterministic BSL structural lint;
- query analysis and non-mutating optimization suggestions;
- fixed-pair configuration/extension comparison;
- profile-local templates and memory;
- EDT remains the authoritative backend for content assist, diagnostics,
  refactoring and project writes.

## Professional analogs

- bulk analysis, local hashed-vector semantic search, symbols and call graph;
- Mermaid architecture and Markdown documentation;
- API compatibility, RLS and exchange-plan inspection;
- YAxUnit alias `run_tests` through the fixed DitriX EDT backend;
- managed external object pipeline `generate_external_object`;
- CLI reports in JSON, SARIF 2.1.0 and HTML.

The deterministic hashed-vector search is deliberately local and dependency
free; it is not claimed to reproduce a proprietary embedding model. Polling
transport without web publication and execution of arbitrary BSL are not
enabled: they require a separately threat-modeled 1C broker and cannot be
implemented safely as a transparent fallback.
